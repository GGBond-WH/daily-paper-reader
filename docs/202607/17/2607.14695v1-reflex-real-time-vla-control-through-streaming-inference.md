## 摘要
流匹配视觉-语言-动作（VLA）模型有望实现精确的连续控制，但其迭代去噪特性与实时机器人控制存在根本性的不兼容：全局时间步注入使KV缓存失效，迫使人们在缓慢的O(N²)重新计算与数学上不正确的缓存重用之间做出选择。我们提出了Reflex框架，该框架利用时间步不变性（即感知编码器在功能上独立于去噪循环），实现了流匹配策略的实时流式推理。Reflex将注意力上下文划分为静态、滑动和动态区域，实现了O(1)增量缓存更新，同时为固定输入保留了等效于整批处理的注意力输出。为了在连续高频推理下保证稳定性，我们引入了AdaRMSNorm，这是一种自适应归一化层，通过根据流阶段进行门控来防止BFloat16数值崩溃。我们进一步通过异步流水线（将视觉编码与动作生成解耦）以及算子融合（减少内核开销）来最大化吞吐量。在LIBERO和Kinetix基准测试中，Reflex实现了2.58倍的推理加速和50Hz的稳定流式传输，将反应延迟降低多达54%，并在不降低性能的情况下实现高效部署。

## Abstract
Flow matching Vision-Language-Action (VLA) models promise precise continuous control, but their iterative denoising nature introduces fundamental incompatibilities with real-time robotics: global timestep injection invalidates KV-caching, forcing a choice between slow $O(N^2)$ re-computation or mathematically incorrect cache reuse. We present \textbf{Reflex}, a framework that enables \textit{real-time streaming inference} for flow matching policies by exploiting the \textit{Timestep-Invariance Property} -- that perception encoders are functionally independent of the denoising loop. Reflex partitions the attention context into static, sliding, and dynamic regions, enabling $O(1)$ incremental cache updates while preserving full-batch-equivalent attention outputs for fixed inputs. To ensure stability under continuous high-frequency inference, we introduce \textit{AdaRMSNorm}, an adaptive normalization layer that prevents BFloat16 numerical collapse by gating on flow phase. We further maximize throughput through an \textit{async pipeline} that decouples visual encoding from action generation, combined with \textit{operator fusion} that reduces kernel overhead. On LIBERO and Kinetix benchmarks, Reflex achieves a 2.58$\times$ inference speedup and 50Hz stable streaming, reducing reaction latency by up to 54\% and enabling efficient deployment without performance degradation.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：流匹配（Flow Matching）视觉-语言-动作（VLA）模型通过迭代去噪生成连续动作，精度高但计算开销大，且其全局时间步嵌入（timestep injection）使得标准KV缓存无效，导致推理延迟与实时控制不兼容。机器人必须以“停止-思考-行动”的阻塞方式执行：观察场景→完整去噪链推理→执行动作。在动态环境中，这种延迟使机器人基于过时信息行动，容易失败。
- **整体含义**：实时高频控制需要50–100Hz的控制频率，而VLA模型单次推理需100–200ms，存在数量级鸿沟。传统加速思路（如加速单个推理步）无法根本解决同步等待问题。Reflex提出了从“加速推理”到“流式异步执行”的范式转变，旨在让机器人边推理边行动，真正实现实时控制。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：利用 **时间步不变性（Timestep-Invariance Property）**——感知编码器（视觉、语言）在功能上独立于去噪步骤t，因此可以固定缓存；而流匹配的迭代部分需要频繁更新。基于此，Reflex将注意力上下文划分为三个区域，实现O(1)增量缓存更新。
- **关键技术细节**：
  - **分区注意力（Partitioned Attention）**：
    - 静态前缀（Static Prefix）：系统指令，计算一次后永久固定在KV缓存中。
    - 滑动历史（Sliding History）：最近N帧观测（如10帧），以FIFO队列维护，内存常数。
    - 动态后缀（Dynamic Suffix）：当前去噪状态x(k)和时间步嵌入，每轮重置。
    - 公式：Attn(x(k)) = Softmax( (Q(k)[K_pin; K_slide(t); K_dyn(k)]^T / √d ) × [V_pin; V_slide(t); V_dyn(k)] )，对于固定输入和固定观测窗口，输出等效于完整整批注意力。
    - 增量预填充（Incremental Prefill）：仅编码最新帧，避免重复计算旧帧。
    - 手动缓存合并（Manual Cache Merging）：在去噪循环前将静态前缀和滑动历史合并为连续内存，动态后缀使用预分配张量，避免动态内存分配。
  - **AdaRMSNorm（自适应根均方归一化）**：
    - 解决BFloat16在高频流式推理中的数值崩溃问题（50Hz操作使模型频繁接触高方差初始化噪声，导致激活值下溢）。
    - 公式：AdaRMSNorm(x, c) = (x / RMS(x)) ⊙ γ(c)，其中γ(c) = 1 + MLP(c)，c = [t_k, s_t]（时间步嵌入+本体感知状态）。RMS计算强制使用FP32，门控MLP使用BFloat16。
    - 稳健数据类型推断（Robust Dtype Inference）自动探测输出投影层精度，避免类型不匹配。
  - **异步流水线（Asynchronous Pipeline）**：
    - 将视觉编码（Thread A，生产者）与策略推理（Thread B，消费者）解耦到两个硬件线程，重叠执行。
    - 未来条件状态预测（Future-Conditional State Prediction）：假设机器人能准确跟踪命令动作，用上次命令动作^a_cmd_t近似未来状态ˆs_{t+Δ}，补偿异步延迟。论文强调这是轻量启发式方法，非形式化等效保证。
  - **系统优化**：
    - 算子融合（Operator Fusion）：融合QKV投影、Gate和Up投影，减少内核启动次数50%，加速15–20%。
    - 静态环形缓冲区（Static Ring Buffer Architecture）：预分配大缓冲区，自定义指针算术，消除动态分配和碎片。
    - 自适应重叠调度（Adaptive Overlap Scheduling）：根据实时推理延迟动态调整重叠K，确保动作块恰好准备好。

## 6. 主要结论与发现

- **推理加速**：Reflex实现2.58×（Pi0.5 on LIBERO）至2.73×（Pi0 on LIBERO）推理加速，且加速比随模型规模增大而增大。

- **稳定性**：AdaRMSNorm使模型在BFloat16下可稳定运行超过2000步（远超典型任务长度），防止了数值崩溃。
- **跨架构与跨硬件泛化**：在Pi0、Pi0.5、SmolVLA上均有类似加速；在RTX 3090上保持2.47×加速比，验证了硬件无关性。
- **操作融合和重叠调度**：分别贡献15–20%和44%的反应延迟降低，是系统级优化的关键。

## 8. 不足与局限

- **实验覆盖的局限性**：
  - 模型仅测试了Pi0系列和SmolVLA，缺乏对更广泛VLA架构（如Octo、RT-2、OpenVLA）的验证。论文承认当前仅适用于感知编码器与时间步无关的架构（如Pi0），对于统一DiT风格（时间步进入视觉编码器）的架构暂不适用。
  - 真实机器人实验仅在一个硬件平台（AgileX PiPer）和三个任务上进行，样本量中等（共180轮）。虽然趋势与仿真一致，但受限于模拟到现实的鸿沟，结果并不代表全面的模拟到现实迁移研究。
  - 基准场景中，LIBERO相对准静态，Kinetix虽动态但任务种类有限。更复杂、更开放的真实操纵场景（如家庭服务、工业装配）未测试。
- **潜在偏差与假设**：
  - 未来状态预测器是有损近似（假设低层控制器准确跟踪命令，且短期动力学可线性化）。论文指出偏差<3cm over 100ms，但更长的预测或更复杂的动力学可能失效。
  - 分区注意力的“等效”声称仅针对“固定观测窗口和固定输入”，异步调度和未来状态预测本身会改变状态条件，因此系统级成功率差异不能归于等效性保证，需结合经验度量。
  - 稳定性实验（表5）基于特定BFloat16实现和硬件，实际中其他数值因素（如量化、内存布局）可能引入新失效模式。
- **应用限制**：
  - 需要模型架构满足感知编码器与时间步解耦的条件，限制了直接迁移到某些扩散Transformer。
  - 50Hz目标控制频率在特定硬件（RTX 4090）上可达，但边缘设备或更低算力平台可能仍需权衡。
  - 方法虽然减少了推理延迟，但未降低模型本身的参数规模和计算复杂度（仍需要2–3B模型），对于资源受限场景仍是挑战。
