## 摘要
参数高效微调减少了模型和优化器内存，但密集注意力仍然使得长训练序列代价高昂。我们将分层全局注意力（HGA）与逐段反向传播和分层KV存储相结合。只有活跃段在显存中保持可微；较早的KV被分离到RAM或NVMe，并且HGA为每个查询块加载一组有界的精确历史词元。在使用4位QLoRA的Qwen3-8B模型和PG19数据集上，16 GB Quadro RTX 5000上的密集训练支持2,048个词元，但在4,096个词元时失败，而HGA在峰值显存15.28 GB下达到16,384个词元。在评估中，同一个适配器在该显卡上运行131,072个词元；显存并非恒定，而是随着驻留块摘要的累积而缓慢增长，因此RAM和NVMe容量设定了超出这些长度的实际限制。在共享的2K训练长度下，在相同的密集注意力读取条件下，HGA训练和密集训练的适配器分别获得2.7405和2.7383 nat，而原始模型获得2.9541 nat。在该边界下，HGA训练已经略快（217.75 vs. 207.02词元/秒），并且HGA与密集的吞吐量比率从1K提高到2K；由于HGA使每个词元关注的固定历史集大致恒定而每个词元的密集工作量增加，我们预计这一领先优势会随着上下文增长而扩大。密集注意力用于主要质量和检索比较，以便它们测量学习到的权重并保持与标准生成框架的兼容性。HGA也可用于检索和生成；一个优化的生产级服务实现正在开发中。

## Abstract
Parameter-efficient fine-tuning reduces model and optimizer memory, but dense attention still makes long training sequences expensive. We combine Hierarchical Global Attention (HGA) with segment-wise backpropagation and tiered KV storage. Only the active segment remains differentiable in VRAM; older KV is detached into RAM or NVMe, and HGA loads a bounded set of exact historical tokens for each query block. On Qwen3-8B with 4-bit QLoRA and PG19, dense training on a 16 GB Quadro RTX 5000 fits 2,048 tokens but fails at 4,096, whereas HGA reaches 16,384 tokens with 15.28 GB peak VRAM. Under evaluation the same adapter runs through 131,072 tokens on this card; VRAM is not constant but grows gently with the resident chunk summaries, so RAM and NVMe capacity set the practical limit beyond these lengths. At the shared 2K training length, HGA-trained and dense-trained adapters obtain 2.7405 and 2.7383 nat under the same dense-attention readout, while the stock model obtains 2.9541. At this boundary HGA training is already marginally faster (217.75 vs. 207.02 tokens/s), and the HGA-to-dense throughput ratio improves from 1K to 2K; because HGA keeps the attended historical set per token approximately constant while dense work per token grows, we expect this lead to widen as context grows. Dense attention is used for the main quality and retrieval comparisons so that they measure the learned weights and remain compatible with standard generation frameworks. HGA can also be used for retrieval and generation; an optimized production-grade serving implementation is under development.

# 有限显存下的长上下文微调：论文总结

## 1. 核心问题与整体含义

长序列微调中，密集注意力（full dense attention）的显存开销随序列长度呈二次增长，成为主要瓶颈。现有参数高效方法（如LoRA、QLoRA）虽然减少了模型权重和优化器状态的显存占用，但注意力计算和反向传播的隐状态（activations）仍需要与序列长度成正比的显存，导致在有限显存（如16GB）下最大训练上下文通常只有2K~4K token。本研究旨在打破这一限制，在有限显存下实现长上下文（如16K甚至更长）的高效微调。

## 2. 方法论

### 核心思想
结合**分层全局注意力（Hierarchical Global Attention, HGA）**、**截断反向传播（TBPTT）** 和**分层KV存储**，将GPU工作集与全量训练历史分离。仅活跃段在VRAM中保持可微，较早的KV被卸载到RAM或NVMe，HGA为每个查询块仅加载有限数量的精确历史token。

### 关键技术细节
1. **上下文分块**：将整个序列划分为固定大小的块（如64 token）和更细粒度的组（如8 token）。
2. **两级摘要**：
   - **块级摘要**：每个64-token块的键（key）的紧凑摘要，常驻VRAM。
   - **组级摘要**：每个8-token组的键摘要，部分缓存在VRAM，缺失时从RAM/NVMe获取。
3. **路由选择**：当前查询块先对块摘要打分，选择最相关的若干块（加上固定的sink和local窗口）；然后在选中的块内再次选择最佳组；最后仅加载选中组的精确token K/V进行注意力计算。路由过程不追踪梯度，K/V收集和精确注意力保持可微。
4. **分段反向传播**：整个序列被分割成固定长度（如2K token）的段。每个段独立进行前向和反向传播，段边界处历史K/V被分离，梯度不跨段传播。
5. **分层KV存储**：
   - L0（VRAM）：常驻的块摘要、活跃段K/V、少量缓存的组摘要。
   - L1（Host RAM）：全量历史K/V、非驻留组摘要。
   - L2（NVMe/文件系统）：冷数据可选溢出层。
6. **显存公式**：
   \[
   M_{GPU} \approx M_{model} + M_{adapters/optimizer} + O(S) + O(B_{route})
   \]
   其中S为段长度，\(B_{route}\)为路由预算。外部历史记录显存开销为\(O(L)\)，但不会增长GPU峰值。

### 算法流程（文字说明）
- 训练时，将长序列划分成多个段（S=2048）。
- 对于每个段：
  1. 计算当前段内所有token的精确K/V。
  2. 利用HGA，从已提交的历史中路由选择一组精确token K/V（块→组→精确token）。
  3. 当前段内的每个query与本地token及选中的历史token进行精确注意力计算。
  4. 计算语言模型损失，立即进行反向传播（仅当前段内参数可微）。
  5. 段处理结束后，将当前段的K/V分离并存储到外部存储器（RAM/NVMe），为下一段腾出VRAM。
- 整个序列的所有段依次处理，完成一次训练步。

### 基准与对比方法
- **基座模型**：Qwen3-8B，4-bit NF4 QLoRA。
- **对比方法**：

- **评估协议**：
  - 主要质量对比使用**密集注意力读取**（dense readout），以确保评估的是学习到的权重而非路由效果。

## 6. 主要结论与发现

1. **训练上下文扩展**：HGA在16GB显存上将可行训练上下文从2K扩展到16K（峰值15.28GB），密集训练在4K即OOM。32K时HGA也OOM（可能因段内显存不够）。
2. **质量等价**：在2K训练长度下，HGA训练与密集训练得到的适配器在密集读取下的损失差异仅为0.0022 nat（HGA: 2.7405 vs Dense: 2.7383），均显著优于基座模型（2.9541）。这表明HGA作为训练时替换不损害质量。
3. **效率优势**：在2K时，HGA已略快于密集训练（217.75 vs 207.02 tok/s），且吞吐量比值从1K的0.80提升到2K的1.05，预计在更长上下文时优势扩大（因为HGA每token的注意力工作量大致恒定，而密集注意力随上下文线性增长）。
4. **检索性能**：两种适配器在RULER检索任务上几乎一致（各仅在96个值中漏掉一个），说明HGA训练不影响密集注意力下的检索能力。
5. **路由稀疏性**：默认8个块已足够好；减少到2块时仍可接受，但PPL增加约0.65；路由质量随块数增加饱和。

## 7. 优点

- **方法创新**：HGA结合TBPTT和分层KV存储，有效分离GPU工作集与全历史，是首次在有限显存下通过精确token路由实现长上下文微调。
- **公平比较**：训练时使用相同数据顺序、种子和超参数；评估时统一采用密集注意力读取，确保对所学权重的公平比较。
- **多维度评估**：从可行性、效率、质量、路由开销、检索能力全面验证，实验设计严谨。
- **开源实现**：代码已在GitHub上公开，可复现。

## 8. 不足与局限

- **因果泄漏（Causal Leakage）**：长期训练（约100M+ token）后会因共享路由决策导致因果泄漏，该方法目前不适合从头预训练，仅适用于微调（文中测试约100步）。
- **实验覆盖有限**：
  - 仅使用单一模型（Qwen3-8B）和单GPU（16GB），缺乏在不同架构、更大模型、更大显存上的验证。
  - 训练步数少（100步），未测试长期微调时的质量稳定性。
  - 长上下文（8K、16K）下缺乏密集训练基线，无法直接比较质量。
- **NVMe后端未基准测试**：虽然实现了文件系统存储，但实验均使用RAM层，未给出NVMe性能数据。
- **HGA推理尚未生产化**：研究阶段的routed推理实现，尚未集成至vLLM/SGLang等标准框架，部署可用性受限。
- **路由开销**：短上下文（1K）时HGA因路由开销比密集慢约20%，仅在较长上下文时才体现优势。
- **TBPTT截断梯度**：虽然历史可被后续段读取，但梯度不能跨段传播，可能影响长距离依赖的学习（但实验表明在有限步数内效果可接受）。
