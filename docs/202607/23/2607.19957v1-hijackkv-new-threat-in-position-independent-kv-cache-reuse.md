## 摘要
键值（KV）缓存可降低大型语言模型（LLM）的推理延迟。传统的基于前缀的缓存在跨推理请求时命中率较低，因为它要求精确的标记和位置匹配。为提高效率，最近的系统优化引入了位置无关的KV复用，允许只要出现相同的文本块，无论其在序列中的位置如何，都可以重用KV缓存。我们发现这种设计引入了一种新威胁——KV缓存劫持。由于KV缓存是通过标记匹配来检索的，但编码了它们最初计算时的上下文，因此与看似良性的标记块绑定的KV可能编码了攻击者控制的前缀。当随后在受害者查询中重用时，即使输入中没有出现攻击者控制的文本，这种受污染的KV也会悄然劫持模型的行为。我们提出了HIJACKKV，这是首个系统性地利用此漏洞的攻击框架，展示了其严重性和实用性。HIJACKKV优化了攻击者控制的前缀，使得后续常见良性文本计算出的KV编码了攻击者的目标，同时该文本在未来缓存命中时保持不变。HIJACKKV在单次尝试中平均成功率达94%，在现实约束下（如10%的低命中率和50%的频繁重计算）仍然有效，能在多轮交互中持续起作用，并在黑盒设置下跨模型迁移。我们还提供了构建安全KV复用系统的设计洞察。

## Abstract
Key-Value (KV) cache reduces inference latency in large language models (LLMs). Traditional prefix-based reuse has low cache hit rates across inference requests because it requires exact token and position matches. To improve efficiency, recent system optimizations introduce position-independent KV reuse, allowing KV cache to be reused whenever identical text chunks appear, regardless of their position in the sequence.   We show this design introduces a new threat, KV Cache Hijacking. Since KV caches are retrieved by token match but encode the context in which they were originally computed, the KV tied to a benign-looking token chunk may encode an attacker-controlled prefix. When later reused in a victim query, this contaminated KV silently hijacks the model's behavior, even if no attacker-controlled text appears in the input.   We introduce HIJACKKV, the first attack framework that systematically exploits this vulnerability, demonstrating its severity and practicality. HIJACKKV optimizes an attacker-controlled prefix, so that the KV computed for a subsequent common benign text encodes the attacker's goal, while the text remains unchanged for future cache hits. HIJACKKV achieves an average 94% success rate in a single attempt, remains effective under realistic constraints including low hit rates (10%) and frequent recomputation (50%), persists over multi-turn interactions, and transfers across models in black-box settings. We further provide design insights for building secure KV reuse systems.

## 1. 核心问题与整体含义（研究动机和背景）
- **研究动机**：大型语言模型（LLM）服务中，KV缓存复用能显著降低推理延迟。传统的基于前缀的复用要求精确的token和位置匹配，命中率低；为此，近期系统优化引入了**位置无关的KV缓存复用**，允许相同的文本块无论出现在何种上下文中均可复用缓存，大幅提高效率。但这种设计忽视了KV状态是上下文相关的：同一文本块在不同前缀下产生的KV状态不同。
- **核心问题**：攻击者可以利用这一特性，通过控制前缀，使得普通文本块的KV状态编码恶意意图；当该KV在受害者查询中被复用时（即使受害者输入中不含任何攻击者控制的文本），模型行为会被无声劫持。论文将此新威胁定义为**KV缓存劫持**。
- **整体含义**：效率优化引入了严重的安全漏洞，传统文本级防御（如提示注入检测）无法防范，因为恶意影响通过内部KV状态而非显式文本传递。论文首次系统性地揭示并利用该漏洞，提出了HIJACKKV攻击框架。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：攻击者构造一个**对抗性前缀**p，将其与一个目标文本块$\tilde{X}$拼接后提交给LLM服务。该前缀p经过优化，使得当LLM计算$\tilde{X}$的KV缓存时，这些KV缓存编码了攻击者期望的恶意目标。之后，受害者发送包含$\tilde{X}$的查询时（$\tilde{X}$可能来自共享知识源如FAQs），位置无关缓存匹配机制会命中攻击者注入的KV，从而生成恶意输出，而受害者输入中没有任何对抗性token。
- **关键技术细节**：
  - **攻击流水线**分两阶段：（1）攻击者本地优化前缀p并提交恶意查询$p \oplus \tilde{X}$；（2）服务端计算KV并存入共享缓存池；受害者查询$X \oplus q$若与$\tilde{X}$匹配，则复用被劫持的KV，得到恶意响应$\tilde{r}$。
  - **优化目标**：寻找前缀$p$，使得在位置无关缓存复用下，LLM输出目标恶意答案$\tilde{r}$。形式化为优化问题：
    \[
    p^* = \arg\min_p \mathcal{L}_{\text{CE}}\left( \text{LLM}(X \oplus q \mid T^p_{\tilde{X}}), \tilde{r} \right)
    \]
    其中$T^p_{\tilde{X}}$是前缀p下$\tilde{X}$的KV缓存。
  - **优化算法**：基于贪心坐标梯度（GCG）算法。主循环迭代更新离散token前缀；在每次迭代中，计算损失相对于one-hot编码前缀的梯度，对每个位置选出top-k候选token，随机采样B个替换候选，并贪心选择使损失最小的更新。
  - **损失计算函数**：模拟完整攻击流水线：将前缀嵌入与目标文本嵌入拼接 → 通过LLM计算KV → 提取目标文本部分的KV → 模拟受害者推理（复用该KV并输入查询q）→ 计算交叉熵损失。
  - 攻击者仅在本地代理模型上运行优化，不接触远程服务端缓存，符合同态威胁模型。

## 3. 实验设计
- **数据集**：四个问答（QA）基准：**HotpotQA**、**SQuAD**（v1.1 & v2.0，通用域）、**MedQA**、**PubMedQA**（医疗域）。每个数据集随机采样200个实例。
- **Benchmark**：每个实例为（问题、真实答案、包含答案的上下文）三元组。攻击目标是为每个问题生成一个特定的错误答案（分类：二选一、多选题则取其他选项；开放题则生成同一类别的错误实体）。
- **对比方法**：
  - **Full**：不使用KV缓存（完全重计算），作为基线。
  - **Vanilla**：完全复用KV缓存（无重计算）。
  - **随机重计算（Random）**：随机选择token重算。
  - **EPIC**：基于注意力下沉效应的重算。
  - **CacheBlend**：选择性重算偏差大的token。
- **默认超参数**：缓存比例$\delta=0.3$，重计算比例$\rho=0.1$，块大小$L_{\text{chunk}}=32$。
- **模型**：Qwen3-8B、Llama-3.1-8B、Ministral-8B（以及迁移实验中的Qwen3-4B/14B、Llama-3.2-1B/3B、Llama-3.3-70B）。
- **指标**：
  - Acc（准确率）：正常回答等于真实答案。
  - U-ASR（非目标攻击成功率）：攻击后输出与正常输出不同。
  - T-ASR（目标攻击成功率）：攻击后输出精确等于攻击者指定的错误答案。

## 6. 主要结论与发现
- **有效性**：HIJACKKV在多种模型和数据集上均达到极高的攻击成功率。例如，在Llama-3.1-8B上，对四种重计算方法的平均T-ASR达89%；在Qwen3-8B上多次接近100%。
- **鲁棒性**：
  - 块大小从32到512变化，T-ASR基本稳定。
  - 缓存比例低至10%时，平均T-ASR仍达67%；比例升至50%时接近100%。
  - 重计算比例从10%升至50%，T-ASR下降但未消除（平均70%→39%），表明重计算不能完全防御。
- **持久性**：在多轮交互中，即使插入1024或2048个无关token后，T-ASR仍保持55%和47%，提示单次注入可长期影响。
- **迁移性**：使用Llama-3.1-8B作为代理模型优化前缀，在黑盒下攻击Qwen3-8B、14B、Llama-3.3-70B等，T-ASR在32%~40%之间（U-ASR更高）；对小模型（1B）T-ASR降至13%，但仍能扰乱输出（U-ASR 90%左右）。
- **消融研究**：
  - GCG优化远优于随机/指令/普通文本前缀。
  - 对释义查询鲁棒（T-ASR 94.0±2.6%）。
  - 在HumanEval代码任务中，多温度下平均ASR 95.5%。
- **防御评估**：
  - 现有防御RobustKV和CachePrune在50%条目剔除后T-ASR仍达79%。
  - 主流缓存压缩方法无法消除恶意影响（平均T-ASR 97%）。
  - 自适应攻击（在优化中引入重计算）能显著提升鲁棒性，跨配置迁移仍有效。
  - 强防御（Hybrid或Attn Score）需重计算80%缓存才能使T-ASR降至11.5%~13%，但计算成本极高。
- **核心结论**：位置无关KV缓存复用存在本质安全缺陷，HIJACKKV能可靠利用；现有启发式防御和压缩技术均不足。

- **方法完整性**：提出了完整的端到端攻击框架（HIJACKKV），包含优化算法、损失函数模拟和本地攻击者模型，无需系统级权限。

## 8. 不足与局限
- **缓存可用性与现实挑战**：攻击成功依赖于被劫持的KV能够在目标用户上下文中被缓存命中且未被驱逐。论文指出存在“流行度权衡”：过于流行的文本可能已有良性缓存，过于冷门的则不易被命中。攻击更适合中等共享度内容。此外，缓存替换策略（FIFO/LRU/LFU）会影响可攻击窗口，但论文未进行真实系统级评估。
- **缓存占用探测**：假设攻击者可根据其目标的缓存状态调整；文中初步探测实验仅用5次查询可达95%识别率，但探测本身可能污染缓存，实际部署需小心预算。
- **黑盒迁移存在差距**：从小模型迁移到大模型时T-ASR尚可，但反向迁移（大→小）时T-ASR大幅下降（仅13%），说明攻击对弱小模型的控制力不足。
- **未测试真实生产环境**：所有实验在受控实验室环境进行，未在多租户在线服务中验证，可能未暴露网络延迟、并发、缓存一致性等实际问题。
- **仅关注LLM生成任务**：仅评估了QA和简单代码任务，未覆盖其他任务类型（如翻译、摘要、对话系统）可能存在的不同表现。
- **计算成本暗示**：GCG优化可能需要大量迭代（默认250步），但论文未报告每个前缀优化所需的具体GPU时间或开销，不利于评估实际攻击可行性。
- **防御建议不够具体**：提出的“混合机制”“偏差拒绝策略”尚属概念，未给出量化安全边界或实现细节，实用性有限。
