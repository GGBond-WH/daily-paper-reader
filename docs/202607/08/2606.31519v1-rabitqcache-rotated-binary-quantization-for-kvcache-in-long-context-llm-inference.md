## 摘要
长上下文大语言模型推理受到大规模键值缓存的严重瓶颈影响，然而现有的稀疏注意力方法通常受限于静态固定预算（Top-k）检索，或依赖于计算成本高昂且有偏的代理分数。为解决这些局限性，我们提出了RaBitQCache，一种新颖的稀疏注意力框架，利用随机旋转二值量化与高吞吐二值-INT4算术高效估计注意力权重。我们的代理分数作为无偏估计器，具有经过证明的误差界，能够实现自适应Top-p检索，根据实际注意力稀疏性动态调整词元预算。我们进一步实现了一个硬件感知系统，通过异步流水线和延迟更新来掩盖开销。评估表明，与最先进的基线相比，RaBitQCache在保持生成质量的同时显著加速了推理并减少了内存I/O。代码可在 https://github.com/Sakuraaa0/RaBitQCache.git 获取。

## Abstract
Long-context Large Language Model inference is severely bottlenecked by the massive Key-Value (KV) cache, yet existing sparse attention methods often suffer from static fixed-budget (Top-k) retrieval or rely on proxy scores that are computationally expensive and biased. To address these limitations, we propose RaBitQCache, a novel sparse attention framework that utilizes randomized rotated binary quantization and high-throughput binary-INT4 arithmetic to efficiently estimate attention weights. Our proxy score serves as an unbiased estimator with a proven error bound, enabling adaptive Top-p retrieval that dynamically adjusts the token budget based on actual attention sparsity. We further implement a hardware-aware system with asynchronous pipelining and lazy updates to mask overhead. Evaluations demonstrate that RaBitQCache significantly accelerates inference and reduces memory I/O while preserving generation quality compared to state-of-the-art baselines. Code is available at https://github.com/Sakuraaa0/RaBitQCache.git.

### 2. 论文提出的方法论：核心思想、关键技术细节

-   **核心思想**：受Johnson-Lindenstrauss引理启发，将高维向量（Key和Query）通过随机旋转和量化映射到一个低精度空间，从而构建一个计算高效的、无偏的、且有严格误差界的内积估计器，用于代理分数计算，并基于此实现自适应Top-p检索。

-   **关键技术细节：**
    1.  **内积分解**：将原始`<q, k>`内积分解为`||q-cq|| * ||k-ck|| * <qc, kc>`与常数项之和，将计算瓶颈锁定在归一化向量的内积`<qc, kc>`。
    2.  **随机旋转二值量化**：
        -   **预填充阶段（索引构建）**：对Key向量进行中心化和归一化后，应用随机旋转矩阵 `P`。然后对旋转后的Key向量进行符号函数量化，生成一个由±1/√D向量构成的二进制码本 `¯Cb`（1比特/维度），并存储一个标量校正因子 `α`。
    3.  **高效的代理分数计算**：
        -   **推理阶段（解码）**：对Query向量进行同样的中心化、旋转操作，并将其进一步量化为4位（INT4）整数向量 `¯qu`。
        -   **核心计算**：将原本昂贵的浮点内积`<qc, kc>` 转化为一个高效的**二进制与INT4点积**运算`<¯Cb, ¯qu>`，该运算基于一个无偏估计公式（`E[<¯kc, qc>/<¯kc, kc>] = <kc, qc>`），且`<¯kc, qc>`可通过INT4点积高效实现。
    4.  **自适应Top-p检索**：由于该估计器是**无偏的**，代理分数在数值上与真实内积可比，因此可以实现基于累积概率的Top-p检索，动态调整Token预算。论文实现了一个高效的、基于线性时间三分搜索的Top-p算子，避免了排序操作。
    5.  **系统级调度优化**：
        -   **异步流水线**：在预填充阶段，将索引构建（量化）与主流的密集注意力计算并行化，从而隐藏索引构建的开销。
        -   **延迟更新**：在解码阶段，新生成的Token暂存在一个全精度的滑动窗口缓冲区中，等到缓冲区满了再批量量化并更新索引，从而减少内核启动开销并利用数据并行性。
    6.  **高效内核实现**：
        -   实现了**INT4 × 二进制GEMV**自定义CUDA内核，利用打包二进制表示、共享内存缓存和向量化位提取等技巧，最大化硬件利用率。
