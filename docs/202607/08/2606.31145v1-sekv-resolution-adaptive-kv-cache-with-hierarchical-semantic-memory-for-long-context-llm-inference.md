## 摘要
大型语言模型越来越多地处理长上下文场景，其中KV缓存成为主要的内存瓶颈：其大小随序列长度线性增长且必须在解码过程中全程保留，导致若不压缩则完整的GPU缓存成本过高。现有的KV缓存压缩方法难以在效率与上下文保真度之间取得平衡。令牌驱逐会丢弃信息，而语义分组在预填充阶段就固定了压缩决策；两者都无法在生成过程中当压缩段变得相关时恢复令牌级别的细节。作为解决方案，我们提出SeKV，一种分辨率自适应的语义KV缓存，它将上下文组织为熵引导的语义段，并将它们存储在GPU-CPU内存层次结构中而不丢弃信息。每个段在GPU上保留一个轻量级摘要向量用于粗略路由，在CPU上保留一个低秩SVD基用于按需令牌级重建。训练得到的缩放机制在解码过程中选择性扩展查询相关的段，无需在GPU上实例化完整KV缓存即可实现精确检索。SeKV实现了自适应的令牌级重建，同时保持基础LLM完全冻结，仅增加少于0.05%的可训练参数。在四个基准测试中，SeKV与最强的语义压缩基线相比平均提升5.9%，同时在128K上下文下与完整KV缓存相比减少53.3%的GPU内存。代码可在https://github.com/AmirAbaskohi/SeKV获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将上下文划分为熵引导的语义跨度，每个跨度以双分辨率存储：GPU保留轻量级摘要向量用于粗粒度路由，CPU保留低秩SVD基用于按需令牌级重建；通过训练的zoom-in机制在解码时动态扩展查询相关的跨度。
- **关键技术细节**：
  - **熵引导跨度分割**：利用预填充阶段的令牌惊奇度（surprisal）`H_t = -log p(x_t | x_<t)`，将惊奇度超过均值+α标准差的位置作为边界，保留这些边界令牌为锚点（anchor tokens）在GPU上全分辨率存储，其余令牌构成语义跨度S。
  - **双分辨率表示**：
    - GPU摘要向量：对每个跨度内的非锚令牌，以惊奇度归一化加权平均，并通过每头每层的可学习投影`W^{(l,h)}`将键向量投影到低维`d'=32`空间，得到路由摘要`\bar{K}_S^{(l,h)}`；同时保留原始空间上的惊奇度加权平均键值对`\bar{k}_S^{(l,h)}`和`\bar{v}_S^{(l,h)}`用于非扩展跨度的粗略注意力。
    - CPU低秩SVD基：对每个跨度S，对键矩阵`K_S^{(l,h)}`进行截断SVD（最大秩R=32），并训练门控预测器`g_ϕ`为每个奇异分量分配软门控`m_{S,i}^{(l,h)}`，有效秩`r_S^{(l,h)}=∑ m`。值矩阵类似处理。存储左奇异向量U、奇异值Σ、右奇异向量V（仅保留有效分量）。短于`L_min=16`的跨度全分辨率保留在GPU。
  - **训练Zoom-in机制**：
    - 路由：每步解码时，将查询投影到路由空间，与每个跨度摘要计算相关性得分`\tilde{α}_S^{(l,h)} = sigmoid( (q·\bar{K}_S^T)/√d' + log|S| )`，并与可学习阈值`τ^{(l,h)}`比较决定是否扩展（二值化通过straight-through estimator）。
    - 注意力混合：锚点始终全分辨率贡献；非扩展跨度贡献一个粗略条目（均值键值对，加log|S|校正）；扩展跨度异步从CPU获取低秩基，重建令牌级键值对参与注意力。预算约束下按路由得分降序选取扩展跨度。
  - **训练目标**：包含四项损失：蒸馏损失（对齐全KV教师）、Zoom损失（正加权二元交叉熵，监督路由概率逼近教师注意力覆盖集）、重建损失（监督门控SVD近似误差）、预算损失（惩罚扩展跨度数和保留的秩，系数β=0.1）。冻结基础LLM，仅训练路由投影`W^{(l,h)}`、阈值`τ^{(l,h)}`和门控预测器`g_ϕ`，总参数量约4.3M（不足0.05%）。

### 8. 不足与局限

- **依赖跨度质量**：熵引导分割可能被高度碎片化文档、表格、代码等干扰，导致次优分配。
- **超参数敏感**：多个设计参数（熵阈值、锚点数量、秩预算、检索预算）需针对新模型或上下文长度调优。
- **依赖CPU-GPU带宽**：多次激活远处跨度时异步传输可能增加延迟；低带宽环境或对抗性查询可能削弱优势。
- **冻结基础LLM**：虽简化部署，但可能限制模型完全适应压缩内存结构。联合训练路由与更灵活重建策略是未来方向。
