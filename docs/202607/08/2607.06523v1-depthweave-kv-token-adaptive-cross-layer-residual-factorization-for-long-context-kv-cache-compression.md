## 摘要
长上下文语言模型推理日益受到存储键值缓存所需内存带宽和容量的限制，然而现有压缩方法通常在各层或各令牌之间应用统一的预算，当词汇线索和语义状态需要不同的保留时，检索性能会下降。我们提出DepthWeave-KV，一种令牌自适应缓存压缩方法，它使用共享低秩通道基对相邻Transformer层的键和值状态进行分解，同时在注意力行为敏感的处保留轻量级的令牌特定残差。DepthWeave-KV将跨深度残差分解与令牌条件深度路由器相结合，为承载指令和检索关键的令牌分配更高的重建秩，并使用来自注意力输出探针的无校准在线错误跟踪，在生成过程中自适应压缩，无需重新训练基础模型。一种融合的CUDA实现联合执行基查找、残差反量化和注意力投影，以减少解码时的内存流量。在LongBench、Needle-in-a-Haystack、L-Eval以及长格式问答和摘要基准测试中，DepthWeave-KV在显著降低内存使用的同时实现了接近完整缓存的任务质量，在先前压缩缓存的基础上提高了平均得分和检索准确率，同时在64K上下文下实现了8.3倍的KV内存减少和每秒72.8个令牌的处理速度。

## Abstract
Long-context language model inference is increasingly limited by the memory bandwidth and capacity required to store key-value caches, yet existing compression methods often apply uniform budgets across layers or tokens and degrade retrieval when lexical cues and semantic states require different preservation. We introduce DepthWeave-KV, a token-adaptive cache compression method that factorizes key and value states across neighboring transformer layers using shared low-rank channel bases while retaining lightweight token-specific residuals where attention behavior is sensitive. DepthWeave-KV combines cross-depth residual factorization with a token-conditional depth router that allocates higher reconstruction rank to instruction-bearing and retrieval-critical tokens, and uses calibration-free online error tracking from attention-output probes to adapt compression during generation without retraining the base model. A fused CUDA implementation jointly performs basis lookup, residual dequantization, and attention projection to reduce decode-time memory traffic. Across LongBench, Needle-in-a-Haystack, L-Eval, and long-form QA and summarization benchmarks, DepthWeave-KV achieves near-full-cache task quality with substantially lower memory use, improving average score and retrieval accuracy over prior compressed caches while reaching 8.3x KV memory reduction and 72.8 tokens per second at 64K context.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：长上下文语言模型推理时，KV（Key-Value）缓存成为主要的内存瓶颈，其巨大的容量需求限制了批处理大小、上下文长度和服务吞吐量。现有压缩方法（如令牌驱逐、合并、量化、低秩近似等）通常对所有层和所有令牌应用**统一压缩预算**，未能考虑到不同层语义角色和不同令牌重要性的差异。这种均匀策略在检索、多跳问答等任务中会导致检索性能严重下降——某些“不重要”的令牌后来可能成为关键检索锚点，但被过度压缩。
- **研究背景**：已有工作（MiniCache、KVSharer）认识到深度维度存在缓存冗余，但直接共享缓存状态仍无法满足异构长上下文负载需求。近期研究也警告KV压缩可能保持平均困惑度但损害检索和指令跟随能力。因此需要一种同时考虑**深度感知（depth-aware）**、**令牌自适应（token-adaptive）** 且能**在线监测注意力行为**的压缩机制。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将相邻Transformer层的KV缓存分解为**共享的低秩通道基**和**令牌特定的残差**。低秩基捕获层间公共结构，残差仅在注意力敏感处保留，且残差秩由令牌条件路由器动态分配。
- **关键技术细节**：
  1. **跨深度残差分解**（Cross-Depth Residual Factorization）：将层划分为重叠深度窗口（$W_m$）。每个窗口内，每个注意力头维护共享基 $\mathbf{B}_{m,h}^K, \mathbf{B}_{m,h}^V \in \mathbb{R}^{r_b \times d}$。重建公式为：
     \[
     \hat{\mathbf{X}}_{\ell,h,t} = \mathbf{A}_{\ell,h,t} \mathbf{B}_{m,h} + g_{\ell,h,t} \mathbf{R}_{\ell,h,t}
     \]
     其中 $\mathbf{A}$ 为系数向量（8-bit块量化），$\mathbf{R}$ 为量化残差（4-bit组量化），$g$ 为轻量残差门控。共享基在预填后通过在线随机低秩草图初始化，并用指数移动平均更新。
  2. **令牌条件深度路由器**（Token-Conditional Depth Router）：为每个令牌计算显著性得分，特征包括：累积注意力质量、最大注意力尖峰、指令/分隔符标记指示、注意力输出重建误差。路由输出离散残差秩（例如 0/2/4/8），使用小规模学习参数 $u$ 和自适应阈值 $\tau_m$。
  3. **无校准在线误差跟踪**（Calibration-Free Online Error Tracking）：每 $p$ 解码步，对部分头进行两次注意力计算（一次用压缩缓存，一次用临时高保真缓存），计算注意力输出相对误差 $\epsilon$。若超过容忍度则降低 $\tau_m$，增加残差秩；否则可放松压缩。
  4. **融合CUDA实现**：将基查找、残差反量化、系数乘法和注意力投影融合为一个核，避免回写重建后的KV张量到全局内存，减少解码时内存流量。
- **算法流程**：预填阶段 → 形成深度窗口基 → 投影到共享系数空间 → 路由器分配残差秩 → 存储量化的残差；解码阶段 → 融合核加载基行 → 反量化路由的残差通道 → 重建KV片段 → 直接参与注意力计算。不修改基础模型权重，仅`g`和路由器参数在一次无标注长文本训练。
