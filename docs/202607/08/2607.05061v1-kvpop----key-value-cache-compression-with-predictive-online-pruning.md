## 摘要
键值（KV）缓存增长是自回归解码中的主要瓶颈，因为内存和带宽随上下文长度线性扩展。现有的KV驱逐方法通常依赖于静态启发式或代理分数，这些方法难以跟踪未来的令牌效用，并且随着相关性的变化导致脆弱的驱逐。为了解决这个问题，我们引入了KVpop，它通过直接监督保留或丢弃决策来学习固定预算的KV驱逐策略。该评分器针对一个新颖的未来注意力目标进行训练，该目标无需生成密集的注意力图即可高效计算。我们还引入了一个基于延迟记忆的评分器，这在学习的驱逐方法中是独一无二的，它将评分延迟固定步数以利用近未来上下文。在AIME和HMMT数学推理中，KVpop在Qwen3-4B上以75%的KV缓存压缩率保留了98%的全注意力性能，在88%压缩率下保留了97%，始终优于已有的驱逐基线。Qwen3-8B显示出更强的结果，达到了接近教师模型的性能。这些结果表明，使用未来注意力信号监督驱逐可以在降低内存成本的同时保持质量。

## Abstract
Key-value (KV) cache growth is a major bottleneck in autoregressive decoding, as memory and bandwidth scale linearly with context length. Existing KV eviction methods often rely on static heuristics or proxy scores, which poorly track future token utility and cause brittle eviction as relevance shifts. To address this, we introduce KVpop, which learns a fixed-budget KV eviction policy by directly supervising the keep-or-drop decision. The scorer is trained against a novel future-attention target, computed efficiently without materializing dense attention maps. We further introduce a delayed memory-based scorer that, uniquely among learned eviction methods, defers scoring for a fixed number of steps to exploit near-future context. On AIME and HMMT mathematical reasoning, KVpop retains 98% of full-attention performance on Qwen3-4B at 75% KV cache compression and 97% at 88% compression, consistently outperforming established eviction baselines. Qwen3-8B shows even stronger results, reaching near-full teacher performance. These results show that supervising eviction with future-attention signals cuts memory costs while maintaining quality.

- **研究动机**：大型语言模型（LLM）自回归解码依赖键值（KV）缓存，但缓存随上下文长度线性增长，成为长序列推理的内存和带宽瓶颈。现有KV驱逐方法（如StreamingLLM、TOVA）依赖静态启发式或在线代理分数，无法准确预测未来token的效用（即一个token在当前不重要但未来可能重要），导致驱逐决策脆弱，尤其是在长程推理场景中。
- **整体含义**：论文提出KVpop，一种**预测性在线剪枝的KV缓存压缩方法**，通过学习直接监督“保留/丢弃”决策，利用**未来注意力信号**作为监督目标，并引入**延迟记忆评分器**在驱逐前积累近未来上下文，从而在固定预算下大幅压缩KV缓存同时保持模型质量。这是在KV驱逐领域中首次直接以未来注意力为目标进行监督学习，并利用延迟评分机制。

### 2. 方法论

- **核心思想**：对每个KV头，维持固定预算 `B = s + w + k`（s个sink token，w个保护窗口中的近邻token，k个长程top-k token）。通过轻量级评分器为每个token分配重要性分数，仅保留得分最高的k个长程token。评分器在训练时被**未来注意力目标**监督，该目标衡量token离开保护窗口后接收到的未来注意力质量。
- **关键技术细节**：
  - **未来注意力目标**：对于每个token t，定义其在保护窗口之后（位置d ≥ t+w）接收的因果注意力概率和的预期值，通过对所有查询头部聚合得到 `r_{h,t}^{tgt}`。该目标无需实例化完整的S×S注意力矩阵，而是通过**转置注意力**（将原keys作为queries，原queries作为keys）并重用稀疏注意力中已有的log-normalizer高效计算。
  - **边界感知损失**：在每个查询位置q，新进入可驱逐集合的token `t_new = q-w` 与教师策略中的截止token `t_bnd` 进行比较。损失是一个成对逻辑损失：`L_score = softplus( -y * (r_hat_new - r_hat_bnd) / τ )`，其中 y 是教师标签（+1保留，-1丢弃）。该损失仅关注边界单对比较，计算复杂度O(1)。
  - **状态评分器（延迟评分）**：引入基于mLSTM的评分器，在token留在保护窗口期间，持续更新mLSTM记忆状态；只有当token离开保护窗口时（即到达驱逐边界），才利用记忆状态进行评分。这允许评分器利用**近未来上下文**（token被评分前的一小段未来上下文）做出更准确的判断。
  - **高效实现**：使用**Fenwick树**（树状数组）维护每个token的静态优先级排名，支持在线查询当前top-k截止排名；配合Triton内核和FlexAttention生成稀疏注意力掩码，无需存储稠密矩阵。
- **算法流程**：训练时，前向传播计算稀疏注意力并得到稀疏LSE；再通过转置注意力计算未来注意力目标；对每个边界对计算边界损失；总损失为KL散度（学生与教师输出）+ 边界损失。仅评分器参数被更新，基础模型参数通过蒸馏更新。

### 7. 优点

- **创新性**：首次使用**未来注意力质量**直接监督驱逐决策，避免启发式代理；引入**延迟评分**机制，利用保护窗口积累无成本上下文。
- **高效性**：训练时的未来注意力目标通过**转置注意力**和重用稀疏LSE计算，避免稠密注意力；推理时仅需一次Fenwick树维护，不增加额外开销。
- **可扩展性**：固定每头预算使得缓存大小恒定，易于执行和编译，缓存布局规整，比动态门方法（如DMS）更高效。
- **泛化性**：仅在数学数据上训练，但在代码和科学任务上保持性能，说明学习到的策略具有一定的领域迁移能力。
- **消融清晰**：通过StreamingLLM+与KVpop的对比，清晰隔离了“训练”和“学习式打分”的不同贡献。

### 8. 不足与局限

- **状态评分器选择单一**：仅评估了mLSTM一种记忆架构，未探索其他线性递归模型（如Mamba、GLA）或更复杂的RNN作为评分器。
- **后训练改装性质**：KVpop是在预训练模型上微调，而非从头训练支持稀疏缓存的架构，可能不是最优的稀疏模型设计。
- **同质化预算**：每个KV头使用相同的固定预算，而论文也指出混合稠密-稀疏层（即部分头保留全部cache）可能进一步提升质量-效率权衡。
- **未在vLLM等分页管理下评估**：论文提到未来工作需测试KVpop在vLLM或SGLang下的优势，当前效率测试仅在简单端到端解码上进行。
- **训练代价**：虽然推理高效，但训练需要同时使用学生和教师模型（教师用于KL蒸馏），并需要额外的转置注意力计算，整体训练成本高于无需教师的方法。
