## 摘要
我们研究了在严格的键值（KV）缓存内存预算下，批量大型语言模型（LLM）推理的非先见调度问题。每个请求有一个已知的提示长度但未知的响应长度，其内存占用包括固定的提示部分以及随每个解码令牌增长的响应部分。在每个解码轮次中，调度器选择一个可行的活动请求批次；驱逐一个请求会丢弃其累积的缓存状态，浪费先前的计算。目标是相对于知道所有响应长度的最优先见调度，最小化总完成时间。我们提出了首个针对任意提示长度和任意响应长度且无额外假设的常数竞争比算法。我们的算法并非依赖单一的通用调度策略，而是基于一种新颖的策略感知路由框架。专门的子调度器处理不同的内存增长几何形态，而一个元调度器在它们之间按时间共享内存预算，并在每个作业的执行过程中动态地根据其行为路由作业。该框架还在在线到达情况下为完工时间和总完成时间提供了常数竞争比保证。

## Abstract
We study non-clairvoyant scheduling for batched Large Language Model (LLM) inference under a hard Key-Value (KV) cache memory budget. Each request has a known prompt length but an unknown response length, and its memory footprint comprises a fixed prompt component together with a response component that grows with each decoded token. At each decoding round, the scheduler chooses a feasible batch of active requests; evicting a request discards its accumulated cache states, wasting prior computation. The goal is to minimize total completion time against the optimal clairvoyant schedule that knows all response lengths.   We present the first constant-competitive algorithm for arbitrary prompt lengths and arbitrary response lengths with no additional assumptions. Rather than relying on a single universal scheduling policy, our algorithm is built on a novel regime-aware routing framework. Specialized sub-schedulers handle different memory-growth geometries, while a meta-scheduler time-shares the memory budget across them and dynamically routes each job as its execution progressively reveals its behavior. This framework also yields constant-competitive guarantees for makespan and for total completion time under online arrivals.

### 2. 论文提出的方法论

- **核心思想**：**Regime-Aware Routing（基于状态的感知路由）**。将作业按提示长度和响应长度的几何特征划分为三个“态”（regime），每个态使用专业化子调度器，并通过一个元调度器在它们之间时间共享内存，同时动态路由（根据执行过程中揭示的响应长度将作业分派到正确的子调度器）。
  - **三个态**：
    - **大型作业（Large jobs）**：提示长度 > Λ（Λ = M/4）。每个作业几乎独自占满内存。
    - **小提示重型作业（Small prompt-heavy jobs）**：提示长度 ≤ Λ 且响应长度 ≤ 提示长度。内存增长不超过提示的常数倍。
    - **小响应重型作业（Small response-heavy jobs）**：提示长度 ≤ Λ 且响应长度 > 提示长度。内存增长远超提示，且面积由响应长度主导。
- **关键技术**：
  - **矩形调度器（Rect，Algorithm 1）**：用于大型和提示重型作业，将其抽象为固定宽度的矩形，采用预算加倍贪心策略（budget-doubling greedy）。通过维护一个按矩形面积预算（width × 2^r）排序的优先队列，并利用队列单调性（queue monotonicity）证明主动尝试的预算始终不大于等待尝试的预算，从而保证完成顺序接近面积顺序。在宽度范围有界（δ ≤ wi ≤ Δ）时，可证明常数竞争比。
  - **响应重型调度器（Response，Algorithm 2）**：利用响应长度为平方的等价关系（Area ~ o^2），采用几何递增的响应上限（τ_r = 2^r）。每阶段使用 Staggered Pipeline Scheduling（SPS）子程序，将所有满足提示长度 ≤ τ_r 的作业视为具有共同代理提示长度（min{τ_r, M-τ_r}）的作业进行批量调度。该子程序能确保高内存利用率，且几何上限保证完成顺序接近非减面积顺序。
  - **元调度器**：
    - **黑盒元调度器（BlackBoxMeta，Algorithm 3）**：已知作业分区时，按倍增阶段轮换运行各子调度器，仅损失常数因子。
    - **路由元调度器（Route，Algorithm 4）**：未知分区时，小作业先进入提示重型分支；若执行到 si 个令牌仍未完成（则被认证为响应重型），再路由到响应重型分支。大作业直接进入大型分支。元调度器使用相同的倍增阶段，确保认证和完成都在常数因子内。
