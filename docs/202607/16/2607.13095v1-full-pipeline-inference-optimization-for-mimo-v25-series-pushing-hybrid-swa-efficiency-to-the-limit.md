## 摘要
我们针对MiMo-V2.5模型系列提出全流水线推理优化方案，该系列结合了混合滑动窗口注意力（Hybrid SWA）、稀疏混合专家（MoE）和多模态编码器。虽然Hybrid SWA理论上能显著减少注意力和KVCache存储，但要在生产环境中实现这些收益需要大量的工程工作。我们系统性地优化了KVCache系统，包括逐层预取、SWA感知前缀缓存树和专门的放置策略，实现了严格的O(W) SWA存储和高缓存命中率。我们进一步构建了GCache，一个具有RDMA优化网络的高性能分布式缓存基础设施，并开发了KVCache亲和路由器，在减少计算的同时保持负载均衡。我们还针对多模态输入进行了优化，包括GPU图像预处理、并行视频解码和多模态缓存共享。这些优化共同构成了首个在生产中高效覆盖Hybrid SWA+MoE+多模态复合架构的大规模LLM服务系统。

## Abstract
We present a full-pipeline inference optimization for the MiMo-V2.5 model family, which combines Hybrid Sliding Window Attention (Hybrid SWA), sparse Mixture-of-Experts (MoE), and multimodal encoders. While Hybrid SWA can ideally reduce both attention compute and KVCache storage significantly compared to Full Attention, realizing these gains in production requires substantial engineering effort. We systematically optimize the KVCache system with layerwise prefetch, SWA-aware prefix cache trees, and specialized placement strategies, achieving strict $O(W)$ SWA storage and high cache hit rates. We further build GCache, a high-performance distributed cache infrastructure with RDMA-optimized networking, and develop a KVCache-affinity router to reduce computation while preserving load balancing. We also optimize for multimodal inputs, including GPU image preprocessing, parallel video decoding, and multimodal cache sharing. Together, these optimizations constitute the first large-scale LLM serving system in production that efficiently covers the Hybrid SWA + MoE + multimodal composite architecture.

## 2. 方法论：核心思想、关键技术细节

### 2.1 KVCache 系统重构

- **双池设计**：将 Full Attention 层和 SWA 层的 KVCache 分离为独立物理池，实现 SWA 存储严格 O(W)（W 为滑动窗口大小 128），容量效率提升约 7 倍。
- **逐层预取**：基于层级别调度，将 GPU 计算与 Host→Device KVCache 预取重叠，使缓存读取开销趋近于零。
- **SWA 感知前缀缓存树**：修改传统 RadixAttention 的匹配规则，引入“窗口安全长度”（tail W tokens 必须有效），避免伪命中导致计算错误。节点携带双索引（Full Attention 段索引 + SWA 段映射），独立管理 SWA 段淘汰。
- **命中率优化**：处理 Device 与 Host 之间 SWA 状态不一致问题（主动补充/异步写入），防止 L3 缓存过早淘汰高频序列，对中短序列保留固定长度 SWA，提升跨会话重用。

### 2.2 GCache：高性能分布式缓存基础设施

- **架构**：去中心化元数据管理（一致性哈希定位）、Master 仅负责心跳和服务发现。支持内存/磁盘两级缓存、共享内存持久化、非阻塞 IO 和 RDMA 通信。
- **网络优化**：优先使用 GPU NIC，NUMA 绑定和 same-rail 亲和策略。1MB IO 单进程 RDMA 读吞吐达 170 GB/s（延迟 280 μs），GDR 场景约 350 GB/s。
- **存储成本优化**：采用共部署（co-deployment）方式，利用 GPU 机器内存和 NVMe SSD，零额外存储成本。
- **可靠性**：单副本容错，通过一致性哈希分散故障影响；硬件故障检测与自动化迁移；超时触发重计算。

### 2.3 调度优化

- **KVCache 亲和路由器**：基于 Radix 前缀树优先将请求调度到已有前缀缓存的 Prefill 节点，同时平衡负载。核心公式：`score(worker) = matchWeight × prefixMatchPercentage - normalizedLoad`。L2 缓存命中率提升约 25%，单节点输入吞吐量提升约 30%。
- **TTFT 优化**：在等待队列中优先调度未缓存 token 少的请求，配合等待时间惩罚避免饥饿，TTFT P90 降低约 30.5%。

### 2.4 预填充优化

- **并行度配置**：SWA 优化后仅需存储窗口内 KVCache，可将专家并行（EP）缩减至原来一半，端到端性能提升约 40%。
- **长度分桶策略**：划分为 0-64K/64K-256K/256K-1M 三类，聚合相似负载的请求，避免长短请求互相干扰。
- **MoE 负载均衡**：预训练已实现良好均衡（平均专家负载因子约 0.85），未额外引入策略。
- **NUMA 冲突解决**：禁用系统内核 `numa_balancing`，性能提升约 10%。

### 2.5 解码优化

- **GPU 内存优化**：SWA 支持使有效 KVCache 容量扩展至 5 倍；PD 分离部署中 KVCache 预分配移至 CPU 内存；优化 CUDA Graph 参数减少浪费。
- **MTP 优化**：预填充阶段引入 MTP 支持并适配 L2/L3 缓存，0-128 token 加速比 2.3×，128-256 加速比 1.5×。

### 2.6 多模态推理优化

- **架构**：多模态嵌入异步传输重叠、Encoder 数据并行（TP=1）、跨请求批处理。
- **预处理**：GPU 图像预处理取代 CPU，并行图像下载/解码，并行视频解码（1 小时视频延迟从 156s 降至 23s），下载与推理工作流解耦。
- **缓存优化**：Encoder 一致性哈希提升缓存命中率 30%，节点内 Embedding 缓存共享。
