## 摘要
设备端大语言模型推理面临响应延迟、有限硬件资源和用户隐私的三难困境。全云推理提供强大的计算能力，但暴露了用户提示和对话数据，而独立设备端推理对于大多数消费类和嵌入式边缘设备而言不可行。本文提出了一种基于端点认证KV缓存的隐私中心型边缘云协同LLM推理框架。本地端点处理输入预处理、嵌入计算、自适应特征优化、KV缓存认证、推测解码和低维模型头部计算，而云端执行认证解码器推理、KV缓存管理、令牌验证和高维词汇投影。端点融合部分输出，应用语言自适应掩码并采样目标令牌。所有传输数据和截断的对数概率均经过量化和AES-GCM加密以保护隐私，核心轻量级模块、草稿参数和缓存访问策略保留在本地以避免泄露。该框架通过优化的流式处理、批处理和量化ONNX部署，支持包括仅CPU、GPU和嵌入式设备在内的异构设备。评估表明，与基线分割推理相比，该框架将每个令牌的延迟降低了高达46.1%，下行链路负载降低了高达67.4%，同时保持了与全云推理相当的性能。

## Abstract
On-device LLM inference faces a trilemma of response latency, limited hardware resources and user privacy. Full cloud inference delivers strong computing power but exposes user prompts and dialogue data, while standalone on-device inference is unfeasible for most consumer and embedded edge devices. This paper presents a privacy-centric edge-cloud collaborative LLM inference framework built on endpoint-authenticated KV cache. Local endpoints handle input preprocessing, embedding computation, adaptive feature optimization, KV cache authentication, speculative decoding and low-dimensional model head calculation, while the cloud conducts authenticated decoder inference, KV cache management, token verification and high-dimensional vocabulary projection. Endpoints fuse partial outputs, apply language-adaptive masking and sample target tokens. All transmitted data and truncated logits are quantized and AES-GCM encrypted for privacy, with core lightweight modules, draft parameters and cache access policies kept local to avoid leakage. The framework supports heterogeneous devices including CPU-only, GPU-equipped and embedded devices via optimized streaming, batching and quantized ONNX deployment. Evaluations demonstrate that the framework reduces per-token latency by up to 46.1\% and downlink payloads by up to 67.4\% over baseline split inference, retaining comparable performance to full cloud inference.

# 高效且隐私感知的大语言模型边缘云协同推理：论文详细总结

## 1. 核心问题与研究动机

**面临的三难困境**：
- 端侧设备（CPU、移动GPU、嵌入式设备）运算能力有限，无法独立运行大语言模型（LLM）。
- 全云推理虽然计算充足，但需要传输原始用户提示和对话历史，造成严重隐私泄露风险。
- 传统分割推理缺乏针对KV缓存的细粒度控制、设备自适应能力以及网络层与隐藏状态层联合保护策略。

**研究目标**：设计一种边缘-云协同LLM推理框架，在保持接近全云推理质量的前提下：
- 降低端到端推理延迟
- 减少传输带宽消耗
- 有效降低隐私暴露（避免原始文本、完整输出分布等外泄）

## 2. 方法论：核心思想与关键技术

### 核心思想
- 将**隐私敏感决策和轻量计算保留在端点**，将**重负载的解码器部分转移到云端**，且云端使用受端点认证的KV缓存。
- 端点控制是否允许云端使用历史KV缓存条目、允许使用多少条目，避免传输原始提示和历史标识。

### 关键技术细节

1. **端侧预处理与特征准备**：
   - 端点计算词嵌入、位置嵌入、类型嵌入，生成隐藏表示 h0。
   - 生成推理控制元组 c = (flag_Cache, L_Cache, d_type, ℓ)，其中 ℓ 为语言标识。
   - 根据设备类型自适应特征分块：CPU用列分块流式传输；GPU用模型切片；边缘设备用ONNX Runtime量化部署。

2. **授权云端解码推理**：
   - 云端接收加密后的特征张量和控制元组，根据 flag_Cache 选择是否使用历史KV缓存（最多 L_Cache 条目）。
   - 执行标准自注意力（含交叉注意力）和前馈网络，返回隐藏状态（可流式返回给CPU端点）。
   - 所有传输张量先量化再AES-GCM加密。

3. **分布式词汇投影（模型头部拆分）**：
   - 将隐藏状态按维度拆分为 endpoints 部分 (D_end) 和 cloud 部分 (D_cloud)。
   - 对应的LM-head权重矩阵也拆分为 W_end 和 W_cloud。
   - 端点计算 z_end = h_end W_end，云端计算 z_cloud = h_cloud W_cloud + b。
   - 端点合并 z = z_end + z_cloud，再通过语言自适应掩码（仅保留 ℓ 语言子集 V_ℓ 的维度）后采样。

4. **端侧推测解码与云端验证**：
   - 端点包含轻量草稿模块 g_ϕ（MLP或小Transformer），基于云端返回的隐藏状态提出 N 个未来令牌。
   - 云端使用目标模型验证草稿令牌：接受则提交并更新KV缓存，拒绝则从第一个拒绝位置开始重算。
   - 草稿长度 N 按设备配置：CPU/边缘设备短草稿，GPU设备长草稿。

5. **传输保护与隐藏状态保护**：
   - 网络层：张量元素量化后经AES-GCM加密传输（密钥预共享，独立随机数）。
   - 端点局部：轻量模块使用LoRA适配，LoRA权重、草稿模块参数、本地投影权重均保留在端点，不传输到云端。
   - KV缓存授权：端点可通过控制元组动态禁用或缩短缓存窗口。

### 算法流程（文字说明）
1. 端点对用户输入x进行分词、嵌入、语言检测，生成控制元组。
2. 端点根据设备类型准备隐藏状态张量（分块/切片/量化），加密后连同控制元组发送至云端。
3. 云端执行授权解码，返回加密后的隐藏状态（或分块）。
4. 对每个生成步骤 t：
   - 端点计算本地投影 z_end[V_ℓ] 并生成草稿令牌序列。
   - 云端计算并返回 z_cloud[V_ℓ]（在授权缓存窗内）。
   - 端点融合对数概率，应用语言掩码，采样下一个令牌 y_t。
   - 端点将 y_t 和草稿序列发送至云端验证。
   - 云端提交接受令牌，更新KV缓存；若拒绝则从首个拒绝位置重算。
5. 重复直至遇到结束符或达到最大长度。
