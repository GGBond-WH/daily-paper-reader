## 摘要
大型语言模型（LLM）智能体积累异构上下文，包括系统指令、计划、用户轮次、检索文档、工具输出和中间推理，其键值（KV）缓存可能成为主要的内存瓶颈。现有的驱逐策略通常对每个令牌应用相同的基于注意力或基于最近性的规则，忽略了智能体编排器已经可用的语义结构。我们提出MemDecay，一种无需训练、区域感知的KV缓存驱逐策略。MemDecay为令牌分配区域特定的基础优先级和衰减率，在令牌获得注意力时刷新保留分数，并在固定缓存预算下驱逐得分最低的页面，同时允许关键区域被固定。我们还提供了一种从测量的注意力寿命中校准衰减率的方法。我们使用Qwen2.5-1.5B和3B模型在约450和1,700个令牌上下文中评估了MemDecay。在所有设置中，不同区域的注意力寿命相差一个数量级：系统令牌的半衰期范围为148到189个解码步骤，而草稿板令牌为14到16个。固定系统区域事实在每个设置中都能以完整缓存精度保留，而没有任何基线能保留超过24个中的13个。随着上下文增长，区域感知的保留仍然有效，而基于最近性的保留则崩溃。然而，累积注意力保留在非固定内容上表现更好，消融实验表明注意力分数归一化是当前公式的主要限制。这些结果确立了语义提示结构作为KV缓存管理的可靠信号，同时阐明了它应与基于注意力的重要性相结合。

## Abstract
Large language model (LLM) agents accumulate heterogeneous context, including system instructions, plans, user turns, retrieved documents, tool outputs, and intermediate reasoning, whose key-value (KV) cache can become a major memory bottleneck. Existing eviction policies generally apply the same attention- or recency-based rule to every token, ignoring semantic structure already available to the agent orchestrator.   We introduce MemDecay, a training-free, region-aware KV-cache eviction policy. MemDecay assigns tokens region-specific base priorities and decay rates, refreshes retention scores when tokens receive attention, and evicts the lowest-scoring pages under a fixed cache budget while allowing critical regions to be pinned. We also provide a procedure for calibrating decay rates from measured attention lifetimes.   We evaluate MemDecay at approximately 450 and 1,700 token contexts using Qwen2.5-1.5B and 3B. Across all settings, attention lifetimes differ by an order of magnitude across regions: system-token half-lives range from 148 to 189 decoding steps, compared with 14 to 16 for scratchpad tokens. Pinning preserves system-region facts at full-cache accuracy in every setting, while no baseline preserves more than 13 of 24. Region-aware retention remains effective as context grows, whereas recency-based retention collapses. Accumulated-attention retention performs better on unpinned content, however, and ablations identify attention-score normalization as the main limitation of the current formulation. These results establish semantic prompt structure as a robust signal for KV-cache management while clarifying how it should be combined with attention-based importance.

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：将提示中的每个令牌按语义区域（如 system、plan、user、tool\_in、tool\_out、retrieval、scratchpad）打标，为每个区域分配基础优先级 br 和衰减率 λr。保留分数结合衰减的结构先验和注意力重要性，加上注意力刷新机制，确保持续使用的令牌保持活跃。
- **关键技术细节**：
  1. **区域标签**：编排器在构造上下文时提供，与连续提示跨度对应，可映射到页面粒度缓存。
  2. **注意力重要性**：仅在每 k 步的观测步骤采样注意力，计算每个令牌接收的平均注意力质量 ¯Ai，用指数加权移动平均更新 ai ← ρ ai + (1−ρ)¯Ai。
  3. **衰减时钟**：从插入时间或最近一次注意力高于阈值 τ 的时间开始计时，得到 Δti(t)。
  4. **保留分数**：si(t) = br(i) exp(−λr(i) Δti(t)) + α ai(t)。br 为初始优先级，λr 控制结构保护衰退速度，α ai 允许注意力保护短期有用的令牌。
  5. **页面级驱逐**：生产服务中KV缓存以固定大小页面组织。页面分数取成员令牌平均。当缓存超出预算 B 时，使用最小堆驱逐得分最低的非固定页面，固定区域（如 system）永不驱逐。
- **算法流程**：每个观测步骤 t：
  1. 更新所有缓存令牌的 ai（式2）；
  2. 更新刷新时钟 tref_i（式3）；
  3. 计算非固定令牌的 si(t)（式4）；

### 6. 论文的主要结论与发现
- **区域寿命层次稳定**：在所有设置中，系统区域半衰期（148–189步）比草稿板（14–16步）高一个数量级，且置信区间不重叠。检索区域是移动区域中寿命最长的。
- **固定指令区域实现无损保留**：MemDecay 固定系统区域后，在所有设置中保持全缓存级别的系统事实召回（24/24或21/24，接近全缓存自身水平），而任何基线最多保留13/24。
- **基于近因的保留随上下文长度崩溃**：streaming在短层尚可，长层接近零；结构先验（区域感知）随上下文增长仍然有效。
- **非固定内容召回弱于累积注意力**：在非系统区域上，H2O风格（累积注意力）表现优于MemDecay（尤其在3B模型上，用户事实召回20/24 vs 5/24）。
- **消融揭示注意力项失效原因**：改变 α 无法补偿；根本原因是页面平均和EWMA平滑使得注意力项幅度远低于结构项，需要幅度归一化（相对于均匀注意力水平）。

### 8. 不足与局限（包括实验覆盖、偏差风险、应用限制）
- **不可逆驱逐**：驱逐后无法恢复，当后续查询使已丢弃令牌变得相关时失效（如用户区域事实因过早衰减而丢失）。
- **跨区域信息流动**：若同一信息出现在不同区域（如工具输出被引用到草稿板），两个副本都快速衰减，可能导致信息丢失。
- **标签依赖**：假设编排器提供标签；无标签时需要分类器，鲁棒性未评估。
- **注意力项幅度问题**：当前公式注意力项贡献太小，无法保护老事实，需归一化设计，而非简单调参。
- **工作负载依赖性**：区域寿命高度依赖具体工作负载（如检索衰减快慢在不同设置中相反），校准需要代表部署场景的数据，通用性受限。
- **实验规模有限**：仅测试1.5B/3B模型、最大1.7k token上下文，未涉及8B+模型、4k+上下文或真实agent轨迹（如SWE-bench、BFCL），结论推广性需谨慎。
- **粒度限制**：每个令牌统一分数，未考虑头/层自适应预算（可正交结合但未实验）。
- **半精度数值影响**：虽然验证了拟合率偏差<0.001%，但长层3B在T4上float16运行，可能引入微小不确定性。
