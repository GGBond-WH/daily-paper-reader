## 摘要
随着大型语言模型（LLMs）处理越来越长的提示，计算和KV-cache内存成本已成为推理系统中的主要瓶颈。现有的输入级提示压缩方法通过标量相关性分数对每个句子进行排序，将文档视为无结构的单词和句子池来解决这一问题。在预算紧张的情况下，这会导致主题坍塌，即文档中的主导主题消耗了预算，丢弃了频率较低但与任务相关的主题。为了保持主题覆盖，需要跨重复出现的主题分配预算，而不是孤立地对句子评分。为此，我们提出了SALT，这是一个与模型无关的抽取式框架，它将每个句子的关键词组织成一个按句子频率（SF）排序的前缀树，句子频率是文档主题结构的轻量级、可重用代理。这种基于前缀树的组织方式平滑了内存分配，并防止主导主题垄断预算。多锚点检索会激活由查询关键词标记的任意深度的前缀树节点，且该前缀树在对话轮次中持续存在，支持多轮使用而无需重新编码文档。通过保留文档主题，SALT减少了长上下文提示的预填充计算和内存成本，同时与针对解码时延和内存的KV-cache方法保持可组合性。

## Abstract
As large language models (LLMs) process increasingly longer prompts, computation and KV-cache memory costs have emerged as major bottlenecks in inference systems. Existing input-level prompt compression methods address this, but rank each sentence by a scalar relevance score, treating the document as an unstructured pool of words and sentences. Under tight budgets, this causes theme collapse, where the dominant theme(s) of a document consumes the budget, discarding less-frequent yet task-relevant themes. Preserving thematic coverage instead requires allocating the budget across recurring themes rather than scoring sentences in isolation. To this end, we propose SALT, a model-agnostic extractive framework that organizes per-sentence keywords into a trie ordered by sentence frequency (SF), a lightweight, reusable proxy for document thematic structure. This trie-based organization smooths memory allocation and prevents dominant themes from monopolizing the budget. Multi-anchor retrieval activates trie nodes labeled by query keywords at any depth, and the trie persists across dialogue turns, supporting multi-turn use without re-encoding the document. By preserving document themes, SALT reduces the prefill computation and memory cost of long-context prompts while remaining composable with KV-cache methods that target decoding-time latency and memory.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：大型语言模型（LLMs）处理越来越长的提示（prompt）时，预填充（prefill）计算量和 KV-cache 内存成为推理系统的关键瓶颈。现有的输入级提示压缩方法（如 RECOMP、EXIT、CPC、Sentinel）通常对每个句子计算一个标量相关性分数，然后按分数高低选择句子填充压缩预算。这种“全局排序+预算填充”的范式在预算紧张时会导致**主题坍塌（theme collapse）**：文档中的主导主题占据大部分预算，而低频但任务相关的次要主题被完全丢弃，影响多跳问答等需要跨主题推理的任务。
- **整体含义**：论文提出将提示压缩重新定义为**在固定预算下保留文档主题覆盖**的问题，而不是孤立地给句子打分。通过显式地按文档中重复出现的词汇主题分配预算，可以避免主题坍塌，同时保持压缩后的文本对下游任务的可用性。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 使用**句子频率（Sentence Frequency, SF）**作为文档主题结构的轻量级代理。SF 指一个关键词出现在多少句子中（作为该句的选定关键词）。高频关键词反映文档的重复主题。
- 构建一个**基于关键词的前缀树（lexical trie）**，其中节点按句子频率排序，叶子节点存储句子ID。预算分配不再基于全局句子分数，而是先在 Trie 的各个主题分支之间分配预算，然后在每个分支内部选择句子。
- 支持两种模式：**摘要模式（summary mode）**——无查询时覆盖全部主题；**查询模式（query mode）**——根据查询关键词激活 Trie 节点，在相关区域内进行覆盖和选择。

### 关键技术细节
1. **索引阶段**：
   - 使用轻量级编码器 BGE-small-en-v1.5，通过 [CLS] 注意力为每个句子中的内容词排序。
   - 对每个句子，按注意力降序添加词，计算当前子集嵌入与全句嵌入的余弦相似度，利用 Kneedle 算法确定保留的关键词数量（上限为句子内容词数的40%），得到关键词集合 `K_i`。
   - 计算每个关键词的句子频率 `SF(w)`，保留高于 `p` 分位数的关键词（默认 `p=0.9`）组成显著集 `S`，其余关键词被剪枝。
   - 对每个句子，取 `T_i = K_i ∩ S`，按 `SF` 降序排列，作为一条路径插入 Trie。路径的根到叶子结点依次存放句子，共享相同前缀的句子聚集在同一分支下。

2. **预算约束的选择阶段**：
   - **摘要模式**：从 Trie 根节点开始，先根据每个一级分支的未覆盖质量（uncovered mass `U_v(R)`）分配预算。分支配额 = 固定底数 + 按 `(M_b + ε)^α` 比例分配的剩余预算（`0<α<1`，用于压缩高支配分支）。然后每个分支内根据句子边际增益公式 `Δ_b(s_i|R)/L(n_i)·I(s_i)` 选择句子，增益基于减少的未覆盖质量。最后用 GlobalFill 分配未使用的预算。
   - **查询模式**：提取查询关键词 `K_q` 和嵌入 `e_q`。激活包含 `K_q` 的 Trie 节点（包括被剪枝但存在于某个 `K_i` 中的关键词）。先进行锚点阶段，选择与查询关键词重叠多且嵌入相似度高的候选句子及其邻居，占用 `β_q B` 预算；剩余预算按摘要模式分配。
   - 选择的句子恢复为文档顺序的纯文本，输入给任意 LLM。

- **Trie 的可重用性**：索引一次后，可在多轮对话中反复使用，且支持不同预算和模式。

## 6. 论文的主要结论与发现

- **主要结论**：SALT 在所有预处理方法中平均准确率最高（LongBench 平均 43.51，领先第二的 RECOMP 约 3 分），且接近最优的 KV-cache 方法（SnapKV 49.04、FastKV 49.23）。在预算紧张时，SALT 通过主题感知分配避免了主题坍塌，从而提高多跳推理和综合理解能力。
- **效率优势**：SALT 的端到端延迟在 256k 输入上为 11.5 秒（H100），显著低于 EXIT（OOM）、CPC（42.3s）、RECOMP（12.0s）、SentenceKV（64.4s）、SnapKV（61.8s），与 FastKV（34.3s）和 DuoAttention（33.4s）相比也更快（因为无需处理完整提示）。
- **内存优势**：峰值 GPU 内存 32.8 GB（256k），低于所有 KV-cache 方法（SnapKV 50.9GB、FastKV 47.7GB），略高于 DuoAttention（31.9GB），但 SALT 无需特殊内核且可在 V100 上运行。
- **多轮成本**：在 19 轮对话中，SALT 累积成本 0.91 秒，远低于无压缩（3.71s）、FastKV（2.62s）和 RECOMP（1.88s），因为索引可复用。
- **硬件兼容性**：SALT 在 V100、A100、H100、B200 上均能运行 256k 上下文，且无需 FlashAttention 等专用内核。
- **检索能力**：NIAH 任务上 SALT 匹配无压缩基线，在超出模型窗口长度时仍能压缩后完成检索。

## 7. 优点：方法或实验设计上的亮点

- **主题覆盖的显式预算分配**：不同于经典 MMR 或子模函数在排序后加入多样性惩罚，SALT 在句子选择之前就将预算分配到主题分支，从根本上防止主导主题贪婪占用预算。这使得低频但关键信息得以保留。
- **轻量且可重用的索引**：使用 BGE-small 编码（33M 参数）构建 Trie，索引一次后可在不同预算、查询模式下反复使用，无需重复编码文档，适合多轮对话和 RAG 场景。
- **模型无关性与兼容性**：输出纯文本，无需修改目标 LLM 架构，可与任何 KV-cache 方法（如 SnapKV、FastKV）叠加使用，同时减少预填充负担。
- **完整的实验覆盖**：不仅评估单轮准确率，还深入测试了端到端效率、多轮累积成本、硬件移植性，并在多个基线上进行公平比较。消融实验直接验证了主题分配的必要性。
- **代码开源**：提供 GitHub 仓库，便于复现和进一步研究。

## 8. 不足与局限

- **代码任务表现差**：所有预处理方法在 LongBench 的 Code 类任务上均低于 KV-cache 方法约 20 分，SALT 也不例外。原因是句子级粒度不匹配代码的 token 级语义，且关键词提取对代码符号（如运算符、标识符）不敏感。
- **句子粒度限制**：SALT 的操作单元是句子，无法像 KV-cache 方法那样做到 token 级剪枝。最紧预算下（<5%）可能无法保留重要短语。论文设置 20% 作为保守下限。
- **依赖外部编码器**：虽然 BGE-small 很轻量，但仍需要一次编码和注意力提取。对于极长文档（>256k）或实时性要求极高的场景，索引开销可能成为瓶颈（但论文已展示 256k 下仍远快于多数基线）。
- **关键词剪枝的敏感性**：默认保留 top 10% 高频关键词（`p=0.9`），若文档主题分散或所有关键词频率都很高，可能丢失一些重要但出现次数少的关键词。论文未深入探讨 `p` 的敏感性。
- **实验未涵盖**：未测试在更大模型（如 70B）上的效果；未与基于 LLM 的在线重写方法（如 LLMLingua 的生成式压缩）对比；未探讨与 KV-cache 方法组合后的准确率变化（仅提及可组合性）。此外，硬件移植仅限 NVIDIA GPU，未在 AMD/Intel 上验证。
- **多轮对话实验的局限性**：使用 50 篇文章的 QuALITY 子集，尚未在更大规模的多轮数据集（如 MultiTurnQA）上验证；且未比较其他预处理方法在多轮下的索引重用（RECOMP 每轮重新编码，但未调整其以支持索引重用）。
