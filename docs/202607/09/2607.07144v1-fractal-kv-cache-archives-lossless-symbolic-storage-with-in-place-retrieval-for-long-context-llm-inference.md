## 摘要
键值（KV）缓存主导了长上下文自回归推理的内存成本，越来越多的研究工作通过量化、驱逐或卸载来压缩它。我们研究一个互补的问题：一旦某个位置的KV状态被量化到码本索引，生成的符号流应该如何存储，并且存储层能否做得比存储更多？本文重新审视了一族压缩迭代映射码，它将符号序列序列化为低维实向量序列，并表明它们为量化KV缓存形成了一种自然的存档格式，具有以下特性：该方法恰好提供了增长缓存所需的访问模式。它是无损的，线性时间运行，并支持O(1)随机访问和O(1)摊还追加。我们在具有1024个token上下文的GPT-2上进行了控制实验，研究向此存档提供输入的量化器。保留一个小的精确窗口（4个注意力缺失点+32个最近的token）并归档其余部分，逐头残差向量量化将存档缓存相对fp16缓存压缩36-54倍，以11-15%的困惑度成本为代价。我们还量化了键/值之间的明显不对称性——量化键的损害大约是量化值的4倍，这与先前的低位KV工作一致——并利用这一点在混合方案中分配比特。最后，我们展示了该存档同时是一个搜索索引：近似子串查询直接在存储的向量上执行，并且从匹配的向量中解码出匹配的上下文，而无需具体化周围的文本。我们发布了所有代码；每个数字都可以在笔记本电脑CPU上通过单个命令复现。

## Abstract
The key-value (KV) cache dominates the memory cost of long-context autoregressive inference, and a growing body of work compresses it through quantization, eviction, or offloading. We study a complementary question: once a position's KV state has been quantized to codebook indices, how should the resulting symbol stream be stored, and can the storage layer do more than store? A family of contractive iterated-map codes that serialize a symbol sequence into a sequence of low-dimensional real vectors is revisited, and it is shown that they form a natural archive format for a quantized KV cache with the following features. The method provides exactly the access pattern a growing cache requires. It is lossless, it runs in linear time, and supports O(1) random access and O(1) amortized append. A controlled study of the quantizer feeding this archive is conducted on GPT-2 with 1024-token contexts. Keeping a small exact window (4 attention sinks + 32 recent tokens) and archiving the rest, per-head residual vector quantization reduces the archived cache by 36-54x relative to an fp16 cache at a perplexity cost of 11-15%, and we quantify a sharp key/value asymmetry -- quantizing keys is roughly 4x more damaging than quantizing values, consistent with prior low-bit KV work -- and use it to allocate bits in a hybrid scheme. Finally, we show the archive is simultaneously a search index: approximate substring queries execute directly on the stored vectors, and matched context is decoded from the matched vector without ever materializing the surrounding text. We release all code; every number reproduces from a single command on a laptop CPU.

## 1. 论文的核心问题与整体含义（研究动机和背景）

长上下文大语言模型（LLM）推理中，KV缓存的内存开销远超模型权重，成为批处理大小和上下文长度的主要瓶颈。现有工作主要通过**有损量化**、**驱逐不重要位置**或**卸载到慢速存储器**来压缩KV缓存。然而，这些方法聚焦于压缩本身，而忽略了**压缩后符号流如何存储**以及**存储层能否提供更丰富的功能**（如随机访问、原地检索）。本文的核心问题是：当KV状态已被量化为一串码本索引（符号流）后，如何设计一种存储格式，使其**同时具备线性时间无损存储、O(1)随机访问、O(1)摊还追加，以及原地检索能力**。作者将一种经典的**分形编码（contractive iterated-map codes）**重新应用于此场景，形成一种“分形KV缓存档案”。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 将量化后的**符号序列**（每个位置由若干码本索引表示）通过一族收缩映射编码为**二维平面上的点序列**。
- 这些点构成一个自相似分形（如Sierpinski垫片），每个符号对应多边形的一个顶点，编码通过迭代映射实现。
- **解码**是编码的逆映射：根据点所在区域确定最后一个符号，然后反向迭代恢复先前符号。
- **存储时**只保留锚点（每隔固定长度存一个点），解码时通过锚点重建局部序列。

### 关键技术细节
- **编码**：给定符号序列 `c1, c2, ..., cL`，初始化点 `p0`（原点），对每个符号 `ck`：
  ```
  pk = V(ck) + r * (p_{k-1} - V(ck))
  ```
  其中 `V(ck)` 是符号 `ck` 对应的多边形顶点，`r` 是收缩比（必须小于等于“kissing常数”以保证无歧义解码）。
- **解码**：给定点 `p`，找到包含它的顶点区域（最近顶点），恢复符号 `ck`，然后反向计算 `p_{k-1}`：
  ```
  p_{k-1} = ( pk - (1-r)*V(ck) ) / r
  ```
- **锚点机制**：由于反向步骤会因浮点精度误差累积，采用固定跨度（span）存储锚点，每个span解码时从锚点向后搜索。
- **作为KV存档**：将每个位置的量化索引（来自残差向量量化）视为符号流，用该编码序列化。支持**O(1)随机访问**（通过定位到包含该位置的锚点，解码该span）和**O(1)摊还追加**（仅重新遍历最后一个锚点之后的符号）。
- **作为检索索引**：由于每个点的坐标主要由最近符号（后缀）决定（几何衰减），两个点的欧氏距离与它们的**公共后缀长度**近似呈指数关系（`2r^s`）。因此，近似子串查询可以直接在存储点上进行最近邻搜索，阈值筛选后解码即可得到匹配上下文。

## 3. 实验设计：数据集、评估基准与对比方法

- **模型**：GPT-2（124M参数）。
- **数据集**：《The Time Machine》（Project Gutenberg，公共领域文本）。
- **评估指标**：**Perplexity**（逐token计算，1024 token上下文窗口）。精确缓存的基准困惑度为**35.97**。
- **实验设置**：
  - 保留**精确窗口**：前4个位置（attention sinks） + 最近32个token。
  - 窗口之外的KV被**量化后存档**，后续token只能访问量化后的缓存。
  - 码本通过K-means在脱敏文本上训练。
- **对比方案**：
  - **Pooled VQ**：所有头共享一个码本（k=256 / 1024）。
  - **Per-head VQ**：每头独立码本（k=256）。
  - **Residual VQ (RVQ)**：多级残差量化（深度为2或更多）。
  - **仅量化值 / 仅量化键**：分析键值不对称性。
  - **混合方案**：键用4级RVQ，值用2级RVQ。
- **压缩比**：以fp16缓存（36,864 B/token）为基准。
- **检索实验**：在100K字符语料上测试近似子串匹配的召回率和精确率。

## 6. 论文的主要结论与发现

1. **分形编码作为KV存档是实用的**：支持线性时间、无损、O(1)随机访问和追加，且可同时作为后缀搜索索引。
2. **Per-head码本优于Pooled码本**：在相同比特预算下，per-head RVQ困惑度损失显著低于pooled（+15.0% vs +37.2%）。
3. **键值不对称性显著**：量化键比量化值造成的困惑度增加大约**4倍**（+14.5% vs +4.0%），且两者近似独立组合。
4. **混合位宽方案最优**：键用4级RVQ、值用2级RVQ，达到**36倍压缩**，困惑度仅增加**11.2%**。
5. **检索功能有效**：基于2D最近邻的近似子串查询召回率为1.0，精度接近1.0（双精度），可直接解码匹配上下文，无需外部文本。

## 7. 优点（方法或实验设计的亮点）

- **存储与检索一体化**：将无损序列化、随机访问、原地检索统一在同一表示中，是新颖的设计。
- **利用键值不对称性进行位宽分配**：定量化键值损伤比，并据此设计混合方案，提升了压缩效率。

- **提出“存档格式”这一概念**，强调存储层可以不只是惰性blob，而是具有主动服务能力。
