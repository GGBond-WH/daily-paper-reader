## 摘要
键值（KV）缓存是长上下文大语言模型（LLM）推理中的主要内存瓶颈。两类领先的无训练方法在结构上都存在局限：令牌选择方法（SnapKV、Ada-KV）从观察窗口中评分重要性并驱逐低分令牌，但驱逐是不可逆的——当重要性信号在查询无关复用下退化时，准确率会下降11-15个百分点；均匀低秩编码保留每个令牌但平均分配秩，浪费了预算。我们观察到这两种失效有一个共同解法：秩应被分配，而非驱逐。我们提出VarRate，一种无训练KV编解码器，根据每个令牌的查询显著性分配可变低秩预算，使每个令牌保持非零秩。类似的自适应秩编解码器只能通过训练实现这种分配；VarRate无需训练。由于不丢弃任何令牌，在查询感知选择会崩溃的情况下，它仅下降3.5-5.5个百分点。在LongBench（16个任务）上匹配20%预算时，VarRate在Llama-3.1-8B和Qwen2.5-7B上均与未压缩模型相差在0.8个点以内。平均两个模型，它是匹配内存下最强的压缩器。在两个模型上均显著优于均匀秩消融。与专门为查询无关复用构建的方法KVzip相比，在四种设置中有三种准确率相当，整体相差在一个点以内，而预填充开销约为其八分之一。

## Abstract
The key-value (KV) cache is the main memory bottleneck in long-context large language model (LLM) inference. Two leading training-free families are both structurally limited: token-selection methods (SnapKV, Ada-KV) score importance from an observation window and evict low-scoring tokens, but eviction is irreversible -- so when the importance signal degrades under query-agnostic reuse, accuracy collapses by 11-15 points; uniform low-rank coding keeps every token but spends equal rank everywhere, wasting budget. We observe that both failures share one cure: rank should be allocated, not evicted. We present VarRate, a training-free KV codec that assigns each token a variable low-rank budget by its query salience, keeping every token at a nonzero rank. Comparable adaptive-rank codecs reach this allocation only through training; VarRate requires none. Because no token is dropped, it degrades by only 3.5-5.5 points where query-aware selection collapses. At a matched 20% budget on LongBench (16 tasks), VarRate stays within 0.8 points of the uncompressed model on both Llama-3.1-8B and Qwen2.5-7B. Averaged over the two, it is the strongest matched-memory compressor. It significantly beats its uniform-rank ablation on both models. Against KVzip, a method purpose-built for query-agnostic reuse, it is accuracy-equivalent in three of four settings and within a point overall, at about one-eighth the prefill overhead.

# VarRate: 训练自由的可变速率 KV 缓存压缩方法——论文详细中文总结

## 1. 核心问题与研究动机

**背景**：长上下文大语言模型（LLM）推理时，KV 缓存随着上下文长度线性增长，成为 GPU 内存的主要瓶颈。压缩 KV 缓存同时保持精度是高效服务的关键问题。

**现有方法局限**：
- **Token 选择方法（如 SnapKV、Ada-KV）**：从观察窗口评分重要性并驱逐低分 token。驱逐不可逆，当缓存被跨查询复用时（查询无关复用），重要性信号过时，导致精度暴跌 11–15 个百分点。
- **均匀低秩编码（如 Palu）**：保留所有 token 但为每个 token 分配相同的低秩预算，忽略 token 的重要差异，浪费预算。
- **量化**：与秩压缩正交，但仅降低数值精度。

**论文动机**：两类失败共享一个解——秩应该被“分配”而非“驱逐”。一个可逆的、分级的秩预算可以让每个 token 保留非零保真度，即使重要性信号过时，误判的 token 只是被粗糙化而非删除。

## 2. 方法论：VarRate

**核心思想**：训练自由的可变速率 KV 编解码器。根据每个 token 的查询显著性（query salience），通过共享低秩基上的“注水”（water-filling）分配可变低秩预算。所有 token 保留至少 `r_min` 秩（非零），不丢弃任何 token。

**关键技术细节**：
1. **预处理**：将每个 post-RoPE 的 key 反旋转为 pre-RoPE 表示，并与 value 拼接成一个 D 维向量 `z_t = [k̃_t; v_t]`，其中 `D = 2·G·d_h`（G 为 KV 头数）。单个基服务所有头。
2. **残差编码**：保留一组步长为 Δ 的锚点 token 和最近 w 个 token 为“精确集”（exact set）；其余 token 编码为与最近 c 个锚点均值的残差：`e_t = z_t - ̄z_t`。
3. **共享基（离线 SVD）**：在少量未标注校准上下文上收集残差，对每层进行 SVD 得到 top-R 右奇异向量 `V ∈ R^{R×D}`。一次计算，无需训练。
4. **显著性评分**：复用 SnapKV 的注意力窗口评分：`s_t = average of attention weights α_{q,t} over observation window`，再经移动平均和 min-max 归一化得到 `ŝ_t ∈ [0,1]`。
5. **注水分配**：在固定总预算 B 下，为每个 coding token 分配秩：
   `r_t = clip(r_min + λ·ŝ_t, r_min, R)`
   乘数 λ 通过迭代注水算法求得，确保 `∑r_t = B`。保证总存储与均匀秩编码器在相同 κ 下完全匹配。
6. **重构**：每个 coding token 投影到共享基 `c_t = V e_t`，按秩截断后重构：`ẑ_t = ̄z_t + V^T (mask ⊙ c_t)`，最后分离 key/value 并恢复 RoPE。

**特点**：无需训练、无需微调、无需架构修改、与量化正交可组合。

## 6. 主要结论与发现

1. **在匹配 20% 预算时**：VarRate 在 Llama 和 Qwen 上分别仅降 0.3 和 0.8 分，优于所有内存匹配的基线。两个模型平均最佳。
2. **显著优于均匀秩消融**：Llama +2.22，Qwen +17.50（p<0.05），在 Qwen 上优势巨大（其 4 头 KV 更难压缩）。
3. **与 KVzip 相当**：准确率方面 3/4 设置无显著差异，整体略低 0.77 分，但预填充开销仅为 1/8（+44% vs +372%）。
4. **查询无关复用下鲁棒**：VarRate 仅降 3.5-5.5 分，而 SnapKV 降 12-15 分，Ada-KV 降 11-15 分。
5. **严格支配 Palu**：在所有预算下优于 Palu，尤其在 20% 预算下 Palu 几乎不可用。
6. **信号和分配均关键**：消融证明两者独立贡献。
7. **与量化正交可组合**：3-bit 量化系数后准确率不变，内存降至 9%。

## 7. 优点

- **无需训练**：节省训练资源和数据标注成本，可直接用于预训练模型。
- **不丢弃 token**：所有 token 保留非零秩，避免不可逆损失。
- **自动适应预算**：注水分配保证存储预算精确匹配均匀秩编码器，便于公平比较。
- **强鲁棒性**：在查询无关复用等困难场景下表现稳定。
- **丰富的消融与验证**：包含信号类型、分配形状、秩底、步长、校准、长上下文、跨模型、跨任务鲁棒性，论证完整。
- **方法论清晰**：从问题分析到设计原则再到实现，逻辑链完整。

## 8. 不足与局限

- **预填充开销**：+44% 预填充时间（相比 +2% 的选择方法和 +24% 的均匀秩方法）。虽远低于 KVzip 的 +372%，但并非零开销。文中的粗略优化（bf16 缓存基等）可降至 +25%，但仍非免费。
- **解码速度**：由于重构全尺寸缓存，解码速度未提升（未使用融合核）。作者指出在部署中需要专用内核才能实现实际加速。
- **预算极紧时（<10% 内存）**：VarRate 可能在共享低秩基的表示能力上受限（如 Qwen 在 8% 预算下崩溃），而 token 选择方法仍可工作。存在运行范围限制。
- **任务类型差异**：在需要分布式信息的长摘要任务中，量化方法（KIVI-4）仍优于 VarRate，因为低秩压缩会丢失稀疏信息，而量化保留所有坐标仅降低精度。
- **校准集依赖性**：离线 SVD 需要少量代表性上下文；尽管实验显示鲁棒，但理论上若校准集极端不匹配，性能可能受影响（未测试极端情况）。
- **未与训练型自适应秩方法（如 MatryoshkaKV、DynaKV）直接对比**：作者承认差异在“性质”上，但未提供公平的内存匹配对比。
- **代码实现的开销**：虽然算法复杂度低，但当前实现中的未融合 fp32 操作较多，仍有优化空间。
